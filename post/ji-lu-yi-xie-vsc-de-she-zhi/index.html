记录一些vsc的设置

2021-08-11 2 min read # settings

1. 如何在markdown中打zzn直接出ZN\mathbb{Z}_NZN​呢

1.1 打开markdown中的quickSuggestion功能

  • ctrl+shift+p 搜索setting.json 或者 打开设置右上角点出json
  • 在setting.json中加入代码
 "[markdown]": {
        "editor.wordWrap": "on",
        "editor.quickSuggestions": true
    },

1.2 给markdown的snippet整个json

  • ctrl+shift+p 搜索snippet 滑到下面找到markdown点它 就会新建一个markdown.json 用于配置markdown语言
  • 示范代码如下
	"zzn": {
		"prefix": "zzn",
		"body": [
			"$\\mathbb{Z}_N = Z/NZ$",
			"$2"
		],
		"description": "ZZN"
	}

1.3 如何处理latex中很多\在json中会当作转义字符

  • 见上面的示例代码 本身应为 $\mathbb{Z}_N = Z/NZ$
    在前面再加个\就可以解决这个问题 这句话本身也运用了这个技巧用以不显示latex而显示源代码

1.4 导出带latex的pdf

Markdown pdf插件能导出，但丑。
Ctrl+Shift+P 后输入 Markdown Pdf 导出为html 再打印 这样出来的Pdf就无法被复制

Markdown One in all 直接出pdf带latex的会有乱码
找到下述地址的配置文件

C://Users/username/.vscode/extensions/yzane.markdown-pdf-1.4.1/template/template.html

/html之前 /body之后加入两句

<script type="text/javascript" src="http://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-AMS-MML_HTMLorMML"></script>
<script type="text/x-mathjax-config"> MathJax.Hub.Config({ tex2jax: {inlineMath: [['$', '$']]}, messageStyle: "none" });</script>

就可以直出pdf了。
注意，上面的网址可能要翻墙，如果导出pdf失败就翻一下
小问题仍然存在，比如 \exist 就编译不出来，解决方法是在vscode右边的预览复制到左边直接用字符
